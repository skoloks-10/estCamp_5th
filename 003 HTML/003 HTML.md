# HTML 텍스트, 목록 및 이미지 태그 종합 가이드

## 1. 텍스트 관련 태그

### 1.1 제목 태그 (`<h1>` ~ `<h6>`)

`<h1>`에서 `<h6>`까지의 태그는 HTML 문서의 제목 계층을 나타내며, 웹 페이지의 논리적 구조를 형성합니다. 숫자가 작을수록 더 중요하고 큰 제목을 의미합니다.

| 태그 | 설명 | 사용 예시 |
| --- | --- | --- |
| `<h1>` | 가장 중요한 제목, 페이지당 한 번만 사용 권장 | `<h1>위니브</h1>` |
| `<h2>` | 주요 섹션의 제목 | `<h2>위니브 캐릭터</h2>` |
| `<h3>` ~ `<h6>` | 하위 섹션의 제목 | `<h3>라이캣</h3>` |

**주의사항:**

- `<h1>` 태그는 페이지당 한 번만 사용 권장
- 스타일링 목적으로 제목 태그를 사용하지 말 것
- 순차적으로 사용해야 함 (예: `<h1>` 다음에 바로 `<h3>`가 오지 않아야 함)

**중요성:**

- 검색 엔진 최적화(SEO)에 중요한 역할
- 스크린 리더 사용자를 위한 페이지 구조 제공
- 페이지의 논리적 구조를 형성

### 1.2 앵커 태그 (`<a>`)

`<a>` 태그는 하이퍼링크를 생성하는 데 사용됩니다. 다른 웹 페이지, 같은 페이지의 특정 부분, 이메일 주소, 전화번호 등으로 연결할 수 있습니다.

| 속성 | 설명 | 예시 |
| --- | --- | --- |
| `href` | 링크 대상의 URL | `<a href="<https://example.com>">예시</a>` |
| `target` | 링크를 열 위치 | `<a href="page.html" target="_blank">새 탭에서 열기</a>` |
| `download` | 링크를 다운로드로 처리 | `<a href="document.pdf" download>PDF 다운로드</a>` |

**href 속성값 유형:**

- 절대 URL: `href="<https://www.example.com>"`
- 상대 URL: `href="about.html"` 또는 `href="/about"`
- 페이지 내 이동: `href="#section1"`
- 전화번호: `href="tel:+1234567890"`
- 이메일: `href="<mailto:example@example.com>"`

**특징:**

- 인라인 요소이지만 예외적으로 블록 요소를 감쌀 수 있음
- `target="_blank"`를 사용할 때는 보안을 위해 `rel="noopener noreferrer"` 추가 권장

### 1.3 문단 태그 (`<p>`)

`<p>` 태그는 하나의 문단을 나타냅니다. 텍스트뿐만 아니라 이미지, 입력 폼 등 관련 콘텐츠를 포함할 수 있습니다.

```html
<p>이것은 문단입니다. 여러 줄의 텍스트를 포함할 수 있으며, 브라우저는 자동으로 줄바꿈을 처리합니다.</p>

```

**특징:**

- 블록 레벨 요소
- 기본적으로 위아래 여백 존재
- HTML에서는 여러 공백과 줄바꿈이 하나의 공백으로 처리됨

### 1.4 줄바꿈 태그 (`<br>`)

`<br>` 태그는 텍스트 내에서 강제 줄바꿈을 생성합니다.

```html
<p>첫 번째 줄<br>두 번째 줄</p>

```

**주의사항:**

- 연속해서 사용하지 말 것 (여백 생성 목적으로 사용 금지)
- CSS `margin`이나 `padding`으로 여백 처리 권장
- 문단 구분에는 `<br>` 대신 `<p>` 태그 사용

### 1.5 줄바꿈 허용 태그 (`<wbr>`)

`<wbr>` 태그는 브라우저가 필요할 때 줄바꿈할 수 있는 위치를 지정합니다. 특히 긴 단어나 URL에 유용합니다.

```html
<p>이것은매우매우매우매우매우매우<wbr>길단어입니다.</p>

```

**특징:**

- 한글에는 `word-break: keep-all` CSS 속성과 함께 사용 필요
- 브라우저 창 크기가 작을 때 지정된 위치에서 줄바꿈 허용

### 1.6 코드 태그 (`<code>`)

`<code>` 태그는 프로그래밍 코드 조각을 표시할 때 사용합니다.

```html
<p>자바스크립트에서 배열에 요소를 추가하려면 <code>push()</code> 메서드를 사용합니다.</p>

```

**특징:**

- 고정폭 글꼴로 표시됨
- 짧은 코드 조각이나 한 줄의 코드에 적합

### 1.7 서식 유지 태그 (`<pre>`)

`<pre>` 태그는 작성한 그대로의 형식을 유지하여 표시합니다. 공백, 탭, 줄바꿈이 그대로 유지됩니다.

```html
<pre>
  function hello() {
    console.log("Hello, World!");
  }
</pre>

```

**특징:**

- 코드 블록이나 ASCII 아트 표시에 유용
- 고정폭 글꼴 사용
- 모든 공백과 줄바꿈을 그대로 표현

### 1.8 강조 태그 (`<strong>`, `<em>`, `<b>`, `<i>`)

| 태그 | 설명 | 예시 |
| --- | --- | --- |
| `<strong>` | 중요하거나 긴급한 콘텐츠 표시 | `<strong>경고!</strong>` |
| `<em>` | 강조하고 싶은 텍스트에 강세 부여 | `<em>매우</em> 중요합니다` |
| `<b>` | 단순 굵은 글씨 처리 (사용 지양) | `<b>굵은 글씨</b>` |
| `<i>` | 일반적인 텍스트와 구분되는 부분 | `<i>외국어 구문</i>` |

**`<strong>` vs `<b>`:**

- `<strong>`은 의미적 중요성을 가지며, 스크린 리더에서 강조하여 읽음
- `<b>`는 단순히 시각적 효과만 있어 의미적 가치 없음

**`<em>` vs `<i>`:**

- `<em>`은 문장의 강세나 강조를 나타내며, 스크린 리더가 다른 억양으로 읽음
- `<i>`는 외국어, 인물의 생각 등 일반 텍스트와 구분이 필요한 부분에 사용

### 1.9 인용 태그 (`<q>`, `<blockquote>`, `<cite>`)

| 태그 | 설명 | 예시 |
| --- | --- | --- |
| `<q>` | 짧은 인라인 인용문 | `<q>인용 내용</q>` |
| `<blockquote>` | 긴 인용문 블록 | `<blockquote>긴 인용 내용</blockquote>` |
| `<cite>` | 저작물 출처 | `<cite>출처 제목</cite>` |

```html
<figure>
  <blockquote>
    <p>"인용된 긴 문장..."</p>
  </blockquote>
  <cite>— 어린왕자 중</cite>
</figure>

```

**특징:**

- `<q>` 태그는 자동으로 인용 부호를 추가함
- `<blockquote>`는 들여쓰기 스타일이 기본 적용됨
- `<cite>` 태그는 출처의 제목을 나타내며 기울임꼴로 표시됨

### 1.10 기타 텍스트 태그

| 태그 | 설명 | 예시 |
| --- | --- | --- |
| `<mark>` | 형광펜 효과 | `<mark>강조 텍스트</mark>` |
| `<dfn>` | 정의 중인 용어 | `<dfn>HTML</dfn>은 마크업 언어입니다` |
| `<abbr>` | 약어 | `<abbr title="Hypertext Markup Language">HTML</abbr>` |
| `<sup>` | 윗첨자 | `x<sup>2</sup>` |
| `<sub>` | 아래첨자 | `H<sub>2</sub>O` |
| `<kbd>` | 키보드 입력 | `<kbd>Ctrl</kbd> + <kbd>S</kbd>` |
| `<address>` | 연락처 정보 | `<address>contact@example.com</address>` |

## 2. 목록 관련 태그

### 2.1 순서가 있는 목록 (`<ol>`)

`<ol>` 태그는 순서가 중요한 목록을 생성할 때 사용합니다.

| 속성 | 설명 | 예시 |
| --- | --- | --- |
| `type` | 목록 항목의 마커 유형 | `<ol type="1">` (1, a, A, i, I) |
| `start` | 시작 번호 | `<ol start="5">` |
| `reversed` | 역순 번호 매기기 | `<ol reversed>` |

```html
<ol type="A" start="3">
  <li>세 번째 항목</li>
  <li>네 번째 항목</li>
  <li>다섯 번째 항목</li>
</ol>

```

**주요 사용 사례:**

- 레시피 단계
- 튜토리얼 지침
- 순위 목록
- 브레드크럼(breadcrumb) 네비게이션

### 2.2 순서가 없는 목록 (`<ul>`)

`<ul>` 태그는 순서가 중요하지 않은 목록을 생성할 때 사용합니다.

```html
<ul>
  <li>커피</li>
  <li>차</li>
  <li>우유</li>
</ul>

```

**주요 사용 사례:**

- 메뉴 항목
- 제품 특징
- 태그 목록
- 관련 링크 목록

### 2.3 목록 항목 (`<li>`)

`<li>` 태그는 목록 항목을 정의하며, `<ol>` 또는 `<ul>` 내에서만 사용할 수 있습니다.

| 속성 | 설명 | 예시 |
| --- | --- | --- |
| `value` | `<ol>` 내에서 항목 번호 지정 | `<li value="10">항목</li>` |

**주의사항:**

- `<li>`는 `<ol>` 또는 `<ul>`의 직접적인 자식으로만 사용 가능
- `<li>` 안에는 다양한 요소가 포함될 수 있음 (텍스트, 이미지, 다른 목록 등)

### 2.4 설명 목록 (`<dl>`, `<dt>`, `<dd>`)

설명 목록은 용어와 그에 대한 설명을 쌍으로 제공하는 목록입니다.

| 태그 | 설명 | 예시 |
| --- | --- | --- |
| `<dl>` | 설명 목록 컨테이너 | `<dl>...</dl>` |
| `<dt>` | 용어(Definition Term) | `<dt>HTML</dt>` |
| `<dd>` | 용어에 대한 설명(Definition Description) | `<dd>웹 페이지 구조화 언어</dd>` |

```html
<dl>
  <dt>위니브</dt>
  <dd>제주도에 위치한 ICT 콘텐츠 회사</dd>
  <dd>개발과 관련된 강의를 주로 진행</dd>
</dl>

```

**특징:**

- 하나의 용어(`<dt>`)에 여러 설명(`<dd>`)이 가능
- 여러 용어(`<dt>`)에 하나의 설명(`<dd>`)도 가능
- 스타일링을 위해 `<div>` 안에 그룹화 가능

**사용 주의사항:**

- 동적으로 변화하는 값을 가진 키-값 쌍에는 적합하지 않음
- 메뉴나 폼에는 사용하지 않음
- 정의되어 있거나 설명이 정해진 콘텐츠에만 사용

## 3. 블록 및 인라인 요소

### 3.1 컨테이너 태그 (`<div>`, `<span>`)

| 태그 | 유형 | 설명 | 예시 |
| --- | --- | --- | --- |
| `<div>` | 블록 | 콘텐츠 분할을 위한 일반 컨테이너 | `<div>콘텐츠 그룹</div>` |
| `<span>` | 인라인 | 텍스트의 일부를 묶는 컨테이너 | `<p>여기 <span>강조</span> 부분</p>` |

**특징:**

- `<div>`는 레이아웃 구성에 주로 사용됨
- `<span>`은 텍스트의 일부에 스타일을 적용할 때 사용됨
- 두 태그 모두 의미적 가치는 없으며 스타일링이나 스크립트를 위한 그룹화에 사용

### 3.2 블록 요소 vs 인라인 요소 vs 인라인-블록 요소

| 특성 | 블록 요소 | 인라인 요소 | 인라인-블록 요소 |
| --- | --- | --- | --- |
| 줄바꿈 발생 | O (세로로 쌓임) | X (가로로 쌓임) | X (가로로 쌓임) |
| 너비/높이 설정 | 가능 | 불가능 | 가능 |
| 여백(margin) | 모든 방향 적용 가능 | 좌우만 적용 가능 | 모든 방향 적용 가능 |
| 패딩(padding) | 모든 방향 적용 가능 | 모든 방향 적용 가능 | 모든 방향 적용 가능 |
| 다른 요소 포함 | 인라인/블록 모두 포함 가능 | 인라인만 포함 가능(a 태그 예외) | - |

**블록 요소 예시:**`<div>`, `<p>`, `<h1>`~`<h6>`, `<ul>`, `<ol>`, `<li>`, `<table>`, `<form>` 등

**인라인 요소 예시:**`<span>`, `<a>`, `<em>`, `<strong>`, `<img>`, `<code>`, `<br>` 등

**인라인-블록 요소 예시:**`<button>`, `<input>`, `<select>`, `<textarea>` 등

## 4. 이미지 관련 태그

### 4.1 이미지 태그 (`<img>`)

`<img>` 태그는 웹 페이지에 이미지를 삽입하는 데 사용됩니다.

| 속성 | 설명 | 예시 |
| --- | --- | --- |
| `src` | 이미지 파일의 경로 | `<img src="image.jpg">` |
| `alt` | 이미지에 대한 대체 텍스트 | `<img src="cat.jpg" alt="고양이 사진">` |
| `title` | 추가 정보(마우스 오버 시 표시) | `<img src="cat.jpg" alt="고양이" title="나의 고양이 모모">` |
| `width`, `height` | 이미지 크기 | `<img src="logo.png" width="200" height="100">` |

```html
<img
  src="character.jpg"
  alt="위니브 검은 토끼 캐릭터 로지"
  width="300"
>

```

**alt 속성 사용 지침:**

- 의미 있는 이미지: 이미지의 내용이나 목적을 설명하는 대체 텍스트 제공
- 장식용 이미지: `alt=""` 사용하여 스크린 리더가 무시하도록 설정
- 절대로 alt 속성을 생략하지 말 것

**파일명 최적화:**

- 검색 엔진은 파일명을 중요한 요소로 활용함
- 의미 있는 파일명 사용 (`img12345.jpg`보다 `black-rabbit-character.jpg` 선호)

### 4.2 이미지 맵 (`<map>`, `<area>`)

이미지 맵은 하나의 이미지 내에서 여러 클릭 가능한 영역을 정의합니다.

```html
<img src="planets.jpg" alt="태양계 행성들" usemap="#planetmap">

<map name="planetmap">
  <area shape="circle" coords="180,139,14" href="mercury.html" alt="수성">
  <area shape="circle" coords="241,139,21" href="venus.html" alt="금성">
  <area shape="rect" coords="290,102,340,176" href="earth.html" alt="지구">
</map>

```

| 속성 | 설명 | 예시 |
| --- | --- | --- |
| `shape` | 영역의 모양 | `shape="rect"`, `shape="circle"`, `shape="poly"` |
| `coords` | 영역의 좌표 | `coords="x1,y1,x2,y2"` (rect), `coords="x,y,radius"` (circle) |
| `href` | 링크 대상 URL | `href="page.html"` |
| `alt` | 영역에 대한 대체 텍스트 | `alt="지구"` |

### 4.3 그림과 캡션 (`<figure>`, `<figcaption>`)

`<figure>` 태그는 이미지, 다이어그램, 코드 등 자체적으로 완결성을 가진 콘텐츠를 그룹화하는 데 사용됩니다. `<figcaption>` 태그는 `<figure>` 요소에 대한 캡션을 정의합니다.

```html
<figure>
  <img src="elephant.jpg" alt="엄마와 아기 코끼리">
  <figcaption>서로를 의지하는 코끼리 가족</figcaption>
</figure>

```

**특징:**

- 본문에서 참조되지만 독립적으로도 이해 가능한 콘텐츠에 적합
- 하나 이상의 이미지, 코드 블록, 다이어그램 등 포함 가능
- `<figcaption>`은 선택적이며, 포함 시 `<figure>` 요소의 첫 번째 또는 마지막 자식으로 배치

**사용 예시:**

- 캡션이 필요한 이미지
- 설명이 필요한 코드 조각
- 데이터 시각화와 함께 제공되는 설명

```html
<figure>
  <pre>
    function hello() {
      console.log("Hello World!");
    }
  </pre>
  <figcaption>간단한 JavaScript 인사 함수</figcaption>
</figure>

```

**img 태그만 사용해야 하는 경우:**

- 단순히 장식용 이미지를 표시할 때
- 본문 흐름의 필수적인 부분으로 이미지가 필요할 때
- 캡션이나 추가 설명이 필요 없을 때

# 최종 요약: HTML 텍스트, 목록 및 이미지 태그

## 텍스트 관련 태그

- **제목 태그(`<h1>-<h6>`)**:
    - 페이지의 계층적 구조를 정의
    - `<h1>`은 페이지당 한 번만 사용 권장
    - SEO와 접근성에 중요
- **앵커 태그(`<a>`)**:
    - 하이퍼링크 생성
    - `href` 속성으로 URL 지정
    - 다양한 리소스(웹페이지, 이메일, 전화번호, 파일 다운로드)로 연결 가능
- **문단 태그(`<p>`)**:
    - 하나의 문단 표현
    - 블록 레벨 요소
- **줄바꿈 태그**:
    - `<br>`: 강제 줄바꿈
    - `<wbr>`: 필요 시 줄바꿈 지점 제안
- **강조 태그**:
    - `<strong>`: 중요성 강조 (굵게 + 의미적 중요성)
    - `<em>`: 강세 부여 (기울임 + 의미적 강조)
    - `<b>`, `<i>`: 시각적 표현만 목적 (권장하지 않음)
- **코드 태그**:
    - `<code>`: 인라인 코드
    - `<pre>`: 형식을 유지한 텍스트/코드 블록
- **인용 태그**:
    - `<q>`: 짧은 인용
    - `<blockquote>`: 긴 인용문
    - `<cite>`: 저작물 출처
- **특수 텍스트 태그**:
    - `<mark>`: 형광펜 효과
    - `<abbr>`: 약어
    - `<dfn>`: 정의 용어
    - `<sup>`, `<sub>`: 윗/아래 첨자
    - `<kbd>`: 키보드 입력
    - `<address>`: 연락처 정보

## 목록 관련 태그

- **순서 있는 목록(`<ol>`)**:
    - 순서가 중요한 항목
    - `type`, `start`, `reversed` 속성으로 커스터마이징
- **순서 없는 목록(`<ul>`)**:
    - 순서가 중요하지 않은 항목
    - 주로 불릿 표시
- **목록 항목(`<li>`)**:
    - `<ol>`이나 `<ul>` 내에서만 사용
    - `value` 속성으로 순서 있는 목록의 번호 지정 가능
- **설명 목록(`<dl>`, `<dt>`, `<dd>`)**:
    - 용어와 정의 쌍으로 구성
    - 용어사전, 메타데이터 표현에 적합
    - 정의가 확실한 콘텐츠에만 사용

## 블록 및 인라인 요소

- **블록 요소**:
    - 전체 너비 차지, 새 줄에서 시작
    - 너비/높이/여백/패딩 설정 가능
    - 예: `<div>`, `<p>`, `<h1>`, `<ul>`
- **인라인 요소**:
    - 콘텐츠 크기만큼만 공간 차지
    - 너비/높이 설정 불가, 상하 여백 적용 불가
    - 예: `<span>`, `<a>`, `<strong>`
- **인라인-블록 요소**:
    - 가로로 배치되나 블록처럼 크기 조절 가능
    - 예: `<button>`, `<input>`

## 이미지 관련 태그

- **이미지 태그(`<img>`)**:
    - `src` 속성으로 이미지 파일 지정
    - `alt` 속성으로 대체 텍스트 반드시 제공
    - 접근성과 SEO에 중요
- **이미지 맵(`<map>`, `<area>`)**:
    - 하나의 이미지에 여러 클릭 영역 생성
    - `shape`, `coords` 속성으로 영역 정의
- **그림과 캡션(`<figure>`, `<figcaption>`)**:
    - 독립적인 콘텐츠 그룹(이미지, 코드 등)
    - 캡션 제공으로 추가 설명 가능
    - 본문과 관련되지만 분리 가능한 콘텐츠에 적합

HTML 태그는 단순히 브라우저 표시를 위한 방법이 아니라, 콘텐츠의 구조와 의미를 명확히 하는 도구입니다. 적절한 시맨틱 태그 사용은 접근성, 검색 엔진 최적화(SEO), 코드 유지보수성을 향상시킵니다.

# HTML 텍스트, 목록 및 이미지 태그 복습 퀴즈

## 문제 1

**웹 페이지에서 가장 중요한 제목을 나타내는 태그로, 페이지당 한 번만 사용하는 것이 권장되는 태그는 무엇인가요?**

A) `<header>`

B) `<h1>`

C) `<title>`

D) `<strong>`

**정답: B) `<h1>`**

**해설:** `<h1>` 태그는 웹 페이지에서 가장 중요한 제목을 나타내며, SEO와 페이지 구조의 명확성을 위해 페이지당 한 번만 사용하는 것이 권장됩니다. `<header>`는 페이지의 소개 및 탐색 영역을 나타내는 시맨틱 태그로, 여러 번 사용 가능합니다. `<title>`은 브라우저 탭에 표시되는 페이지 제목을 정의하는 태그이며, `<strong>`은 중요한 텍스트를 강조하는 인라인 태그입니다.

## 문제 2

**다음 중 `<a>` 태그의 속성으로, 링크를 클릭했을 때 파일을 다운로드하도록 지시하는 속성은?**

A) `href`

B) `target`

C) `download`

D) `link`

**정답: C) `download`**

**해설:** `download` 속성은 `<a>` 태그에 사용되어 링크를 클릭했을 때 페이지 이동 대신 URL의 콘텐츠를 다운로드하도록 브라우저에 지시합니다. `href` 속성은 링크의 대상 URL을 지정하고, `target` 속성은 링크를 어디에 열지 지정하며(예: `_blank`는 새 탭), `link`는 `<a>` 태그의 유효한 속성이 아닙니다.

## 문제 3

**다음 중 순서가 있는 목록(`<ol>`)의 속성으로, 목록 항목의 번호 매김을 역순으로 만드는 속성은?**

A) `reverse`

B) `reversed`

C) `descending`

D) `order="desc"`

**정답: B) `reversed`**

**해설:** `reversed` 속성은 `<ol>` 태그에서 목록 번호를 역순으로 표시하게 합니다. 이 속성은 불리언 속성으로, 단순히 존재 여부만으로 효과를 발휘합니다. 예를 들어 `<ol start="5" reversed>`는 5부터 시작해서 카운트다운하는 목록을 만듭니다. `reverse`, `descending`, `order="desc"`는 HTML에서 `<ol>` 태그의 유효한 속성이 아닙니다.

## 문제 4

**인용문을 나타내는 두 태그 `<q>`와 `<blockquote>`의 주요 차이점은 무엇인가요?**

A) `<q>`는 자동으로 인용부호를 추가하지만, `<blockquote>`는 추가하지 않는다

B) `<q>`는 짧은 인라인 인용에 사용하고, `<blockquote>`는 긴 블록 인용에 사용한다

C) `<q>`는 의미적 가치가 있고, `<blockquote>`는 단순 스타일링 용도이다

D) `<q>`는 여러 문단을 포함할 수 있지만, `<blockquote>`는 한 문단만 포함할 수 있다

**정답: B) `<q>`는 짧은 인라인 인용에 사용하고, `<blockquote>`는 긴 블록 인용에 사용한다**

**해설:** `<q>` 태그는 줄바꿈 없이 텍스트 흐름 내에서 사용되는 짧은 인용문에 적합하며, 브라우저가 자동으로 인용부호를 추가합니다. 반면 `<blockquote>` 태그는 별도의 블록으로 표시되는 긴 인용문에 적합하며, 일반적으로 들여쓰기 스타일이 적용됩니다. 두 태그 모두 인용문을 나타내는 의미적 가치를 가지며, `<blockquote>`도 여러 문단을 포함할 수 있습니다.

## 문제 5

**다음 중 이미지의 접근성을 위해 가장 중요한 속성은?**

A) `src`

B) `alt`

C) `title`

D) `width`

**정답: B) `alt`**

**해설:** `alt` 속성은 이미지의 대체 텍스트를 제공하여 스크린 리더 사용자가 이미지 내용을 이해할 수 있게 하고, 이미지 로딩에 실패했을 때도 정보를 제공합니다. 따라서 웹 접근성에 매우 중요한 속성입니다. `src`는 이미지 파일의 위치를 지정하는 필수 속성이지만 접근성과는 직접적인 관련이 없고, `title`은 마우스 오버 시 나타나는 툴팁 텍스트로 접근성의 핵심 요소가 아니며, `width`는 이미지 너비를 지정하는 시각적 속성입니다.

## 문제 6

**설명 목록을 만드는 태그 조합으로 올바른 것은?**

A) `<dl>`, `<dt>`, `<dd>`

B) `<dl>`, `<li>`, `<desc>`

C) `<dlist>`, `<term>`, `<desc>`

D) `<list>`, `<term>`, `<definition>`

**정답: A) `<dl>`, `<dt>`, `<dd>`**

**해설:** HTML에서 설명 목록(Description List)은 `<dl>` 태그로 생성하며, 이 안에는 `<dt>`(Definition Term, 용어)와 `<dd>`(Definition Description, 설명) 태그가 포함됩니다. 이 구조는 용어사전, 메타데이터 등 용어와 그에 대한 설명을 쌍으로 제공하는 데 적합합니다. 나머지 선택지들은 유효한 HTML 태그 조합이 아닙니다.

## 문제 7

**다음 중 블록 요소와 인라인 요소의 차이점으로 올바르지 않은 것은?**

A) 블록 요소는 새 줄에서 시작하지만, 인라인 요소는 현재 줄에 이어서 표시된다

B) 블록 요소는 너비와 높이 속성을 설정할 수 있지만, 인라인 요소는 불가능하다

C) 블록 요소는 상하 마진을 가질 수 있지만, 인라인 요소는 좌우 마진만 가질 수 있다

D) 블록 요소는 인라인 요소를 포함할 수 없지만, 인라인 요소는 블록 요소를 포함할 수 있다

**정답: D) 블록 요소는 인라인 요소를 포함할 수 없지만, 인라인 요소는 블록 요소를 포함할 수 있다**

**해설:** 블록 요소는 인라인 요소를 포함할 수 있습니다. 반면, 인라인 요소는 일반적으로 블록 요소를 포함할 수 없습니다(단, `<a>` 태그는 예외적으로 블록 요소를 포함할 수 있음). 나머지 설명은 모두 정확합니다: 블록 요소는 새 줄에서 시작하고, 너비/높이 설정이 가능하며, 상하/좌우 마진을 모두 가질 수 있습니다. 인라인 요소는 줄바꿈 없이 콘텐츠 흐름을 따라가며, 너비/높이 설정이 불가능하고, 좌우 마진만 적용 가능합니다.

## 문제 8

**다음 중 텍스트에 강조를 표현하기 위한 태그로, 의미적으로 중요성을 나타내고 스크린 리더가 다른 억양으로 읽도록 하는 태그는?**

A) `<b>`

B) `<i>`

C) `<em>`

D) `<mark>`

**정답: C) `<em>`**

**해설:** `<em>` 태그는 텍스트를 강조하기 위해 사용되며, 시각적으로는 기울임꼴로 표시되지만 의미적으로는 강세나 강조를 나타냅니다. 스크린 리더는 이를 다른 억양으로 읽어 강조의 의미를 전달합니다. `<b>` 태그는 단순히 시각적으로 굵은 글씨체를 적용하고, `<i>` 태그는 일반 텍스트와 구분되는 부분(외국어, 기술 용어 등)을 표시하며, `<mark>` 태그는 형광펜과 같이 텍스트를 하이라이트하는 용도로 사용됩니다.

## 문제 9

**웹 브라우저에서 공백, 탭, 줄바꿈을 모두 그대로 표시해주는 태그는?**

A) `<p>`

B) `<pre>`

C) `<code>`

D) `<span>`

**정답: B) `<pre>`**

**해설:** `<pre>` 태그(Preformatted Text)는 HTML 코드에 작성된 그대로의 형식을 유지하여 표시합니다. 즉, 모든 공백, 탭, 줄바꿈이 그대로 브라우저에 표시됩니다. 이 태그는 코드 조각, ASCII 아트, 사전 형식화된 텍스트 등을 표시할 때 유용합니다. `<p>` 태그는 단락을 나타내며 공백과 줄바꿈을 하나의 공백으로 축소합니다. `<code>` 태그는 고정폭 글꼴로 표시되지만 공백과 줄바꿈을 보존하지 않습니다. `<span>` 태그는 인라인 요소로 특별한 형식화 기능이 없습니다.

## 문제 10

**다음 중 `<figure>` 및 `<figcaption>` 태그의 올바른 사용법은?**

A) 페이지의 모든 이미지는 반드시 `<figure>` 태그 안에 넣어야 한다

B) `<figcaption>`은 `<figure>` 밖에 위치하여 여러 그림에 대한 설명을 한꺼번에 제공한다

C) `<figure>`는 이미지와 그에 대한 캡션을 연결하여, 독립적으로 참조할 수 있는 콘텐츠를 묶는다

D) `<figcaption>`은 여러 개 사용하여 하나의 `<figure>` 안에 여러 설명을 제공해야 한다

**정답: C) `<figure>`는 이미지와 그에 대한 캡션을 연결하여, 독립적으로 참조할 수 있는 콘텐츠를 묶는다**

**해설:** `<figure>` 태그는 이미지, 도표, 코드 등 독립적으로 참조할 수 있는 콘텐츠를 그룹화하는 데 사용됩니다. `<figcaption>` 태그는 해당 `<figure>` 요소에 대한 캡션을 제공하며, `<figure>` 안에 첫 번째 또는 마지막 자식 요소로 위치해야 합니다. 모든 이미지가 반드시 `<figure>` 안에 들어갈 필요는 없으며(장식적 이미지 등), `<figcaption>`은 하나의 `<figure>` 당 하나만 사용하는 것이 올바른 사용법입니다.
