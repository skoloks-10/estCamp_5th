# 웹 접근성과 시맨틱 HTML 종합 가이드

## 1. 웹 접근성(Web Accessibility)

### 1.1 웹 접근성의 개념

웹 접근성이란 장애인, 고령자 등 모든 사용자가 웹에서 제공하는 정보와 기능에 동등하게 접근할 수 있도록 보장하는 것입니다. 웹의 창시자 팀 버너스 리는 "웹의 힘은 그것의 보편성에 있다. 장애에 구애없이 모든 사람이 접근할 수 있는 것이 필수적인 요소이다"라고 말했습니다.

### 1.2 웹 접근성의 중요성

- **사회적 포용성**: WHO에 따르면 전 세계 인구의 약 15%(약 13억 명)가 심각한 장애를 경험하고 있습니다.
- **법적 의무**: 많은 국가에서 웹 접근성은 법적 요구사항입니다. 한국에서는 2013년 4월부터 장애인차별금지법에 따라 웹 접근성 준수가 의무화되었습니다.
- **사용자 경험 향상**: 접근성이 높은 웹사이트는 모든 사용자에게 더 나은 사용자 경험을 제공합니다.

### 1.3 정보에 접근하는 다양한 방법

| 장애 유형 | 접근 방법 | 설명 |
| --- | --- | --- |
| 시각 장애(전맹) | 스크린 리더 | 화면의 텍스트를 음성으로 읽어주는 프로그램 |
| 시각/청각 장애 | 점자 정보 단말기 | 손으로 읽을 수 있는 점자 디스플레이 |
| 시각 장애(저시력) | 화면 확대 | 화면 콘텐츠를 확대하여 보기 쉽게 함 |
| 시각 장애 | 색 전환(고대비) | 배경과 텍스트 간의 대비를 높여 가독성 향상 |
| 지체 장애 | 특수 입력 장치 | 헤드 포인터, 빅키 키보드, 트랙볼 마우스 등 |

## 2. 웹 콘텐츠 접근성 가이드라인(WCAG)

### 2.1 WCAG의 정의

WCAG(Web Content Accessibility Guidelines)는 W3C에서 제공하는 웹 콘텐츠 접근성 표준 권고안으로, 웹사이트와 애플리케이션이 장애인들에게 접근 가능하도록 만들기 위한 지침입니다.

### 2.2 WCAG 4대 원칙

| 원칙 | 설명 | 예시 |
| --- | --- | --- |
| 인식 가능(Perceivable) | 모든 사용자가 콘텐츠를 인식할 수 있어야 함 | 이미지에 대체 텍스트 제공, 자막 제공 |
| 운용 가능(Operable) | 모든 사용자가 서비스의 기능을 조작할 수 있어야 함 | 키보드로 모든 기능 접근 가능, 충분한 시간 제공 |
| 이해 가능(Understandable) | 모든 사용자가 콘텐츠와 인터페이스를 이해할 수 있어야 함 | 명확한 지침과 피드백, 일관된 탐색 |
| 견고함(Robust) | 다양한 기기 및 브라우저에서 접근 가능해야 함 | 표준 준수, 다양한 보조 기술과 호환 |

### 2.3 WCAG 준수 수준

| 수준 | 설명 |
| --- | --- |
| A (최소) | 기본적인 접근성 요구사항을 충족, 이를 충족하지 않으면 웹사이트 사용이 불가능할 수 있음 |
| AA (표준) | 대부분의 접근성 규정에서 요구하는 수준, 합리적인 접근성 보장 |
| AAA (최적) | 최고 수준의 접근성 제공, 모든 사용자에게 평등한 웹 경험을 제공 |

### 2.4 한국형 웹 콘텐츠 접근성 지침(KWCAG)

KWCAG는 WCAG를 기반으로 한국 실정에 맞게 수정한 접근성 지침입니다. 주요 검사 항목에는 대체 텍스트 제공, 키보드 접근성, 충분한 시간 제공, 콘텐츠 명확성, 예측 가능성, 호환성 등이 포함됩니다.

### 2.5 간단한 접근성 확인 방법

| 방법 | 설명 | 확인 포인트 |
| --- | --- | --- |
| 탭 키 사용 | 키보드로 모든 요소 접근 가능 여부 확인 | 모든 상호작용 요소에 순차적으로 접근 가능해야 함 |
| 포커스 확인 | 현재 포커스된 요소 구분 가능 여부 | 포커스 상태가 시각적으로 명확히 표시되어야 함 |
| 호버 스타일 | 마우스 호버 시 변화 확인 | 색상 외에도 다른 시각적 표시가 있어야 함 |
| 접근성 도구 | Lighthouse, WAVE 등 활용 | 자동화된 도구로 기본적인 접근성 문제 감지 |

## 3. 시맨틱 HTML

### 3.1 시맨틱 마크업의 중요성

시맨틱 마크업은 HTML 요소에 의미를 부여하여 웹 페이지의 구조와 콘텐츠를 명확하게 정의합니다. 이는 접근성, 검색 엔진 최적화(SEO), 코드 유지보수성을 향상시킵니다.

### 3.2 주요 시맨틱 요소

| 요소 | 설명 | 사용 예시 |
| --- | --- | --- |
| `<header>` | 소개 및 탐색 정보 (로고, 제목, 검색 폼 등) | 페이지 상단이나 섹션의 시작 부분 |
| `<nav>` | 주요 탐색 링크 모음 | 메뉴, 목차, 브레드크럼(경로) 등 |
| `<main>` | 페이지의 주요 콘텐츠 (페이지당 한 번만 사용) | 본문 내용 |
| `<article>` | 독립적으로 배포/재사용 가능한 콘텐츠 | 뉴스 기사, 블로그 포스트, 위젯 |
| `<section>` | 주제별로 그룹화된 콘텐츠 | 장, 탭 콘텐츠, 번호가 매겨진 섹션 |
| `<aside>` | 주요 콘텐츠와 간접적으로 관련된 콘텐츠 | 사이드바, 광고, 관련 링크 |
| `<footer>` | 저작권 정보, 연락처 등 페이지 하단 정보 | 페이지나 섹션의 끝 부분 |

### 3.3 시맨틱 요소 사용 예시

```html
<header>
  <h1>회사명</h1>
  <nav>
    <ul>
      <li><a href="#">홈</a></li>
      <li><a href="#">소개</a></li>
      <li><a href="#">제품</a></li>
      <li><a href="#">연락처</a></li>
    </ul>
  </nav>
</header>

<main>
  <article>
    <h2>최신 뉴스</h2>
    <p>뉴스 내용...</p>
  </article>

  <section>
    <h2>제품 소개</h2>
    <p>제품에 대한 설명...</p>
  </section>

  <aside>
    <h3>관련 링크</h3>
    <ul>
      <li><a href="#">관련 기사 1</a></li>
      <li><a href="#">관련 기사 2</a></li>
    </ul>
  </aside>
</main>

<footer>
  <p>&copy; 2025 회사명. All rights reserved.</p>
</footer>

```

### 3.4 비교: article과 section

| 특성 | article | section |
| --- | --- | --- |
| 독립성 | 독립적으로 존재할 수 있는 콘텐츠 | 관련된 콘텐츠의 묶음 |
| 재사용성 | 다른 맥락에서 재사용 가능 | 특정 맥락에 종속됨 |
| 사용 예 | 블로그 포스트, 뉴스 기사, 댓글 | 장, 탭 패널, 그룹화된 콘텐츠 |
| 제목 요소 | 제목 요소(`<h1>-<h6>`)를 포함해야 함 | 제목 요소를 포함해야 함 |

**사용 지침**: 콘텐츠가 독립적으로 의미가 있다면 `<article>`, 문서의 구조적 분리가 필요하면 `<section>`, 단순 스타일링 목적이면 `<div>`를 사용합니다.

### 3.5 헷갈리기 쉬운 태그 비교

| 비교 태그 | 차이점 |
| --- | --- |
| `<i>` vs `<em>` | `<i>`: 단순 이탤릭체 표시, `<em>`: 강조의 의미 포함 |
| `<em>` vs `<strong>` | `<em>`: 일반 강조, `<strong>`: 강한 중요성/긴급성 강조 |
| `<img>` vs CSS background | `<img>`: 콘텐츠의 일부, `background`: 장식 목적 |
| `<button>` vs `<a>` | `<button>`: 동작 실행, `<a>`: 페이지/위치 이동 |
| `<table>` vs CSS grid | `<table>`: 표 형식 데이터, CSS grid: 레이아웃 배치 |

## 4. HTML 표(Table) 요소

### 4.1 기본 표 구조

| 요소 | 설명 | 예시 |
| --- | --- | --- |
| `<table>` | 표 전체 컨테이너 | `<table>...</table>` |
| `<tr>` | 표 행(row) | `<tr>...</tr>` |
| `<td>` | 표 데이터 셀 | `<td>데이터</td>` |
| `<th>` | 표 제목 셀 | `<th>제목</th>` |
| `<caption>` | 표 설명/제목 | `<caption>표 제목</caption>` |

### 4.2 표 구조화 요소

| 요소 | 설명 | 사용 시기 |
| --- | --- | --- |
| `<thead>` | 표 헤더 그룹 | 제목 행 그룹화 |
| `<tbody>` | 표 본문 그룹 | 데이터 행 그룹화 |
| `<tfoot>` | 표 푸터 그룹 | 요약 행 그룹화 |
| `<colgroup>` | 열 그룹 | 열 단위 스타일 적용 |
| `<col>` | 개별 열 | 특정 열 스타일 적용 |

### 4.3 표 접근성 향상 속성

| 속성 | 설명 | 사용 예시 |
| --- | --- | --- |
| `scope` | 제목 셀의 범위 지정 | `<th scope="col">이름</th>` |
| `colspan` | 여러 열 병합 | `<td colspan="2">병합된 셀</td>` |
| `rowspan` | 여러 행 병합 | `<td rowspan="3">병합된 셀</td>` |
| `id`-`headers` | 복잡한 표에서 셀 관계 정의 | `<th id="name">이름</th><td headers="name">홍길동</td>` |

```html
<table>
  <caption>2025년 분기별 매출</caption>
  <colgroup>
    <col class="quarter">
    <col span="3" class="data">
  </colgroup>
  <thead>
    <tr>
      <th scope="col">분기</th>
      <th scope="col">서울</th>
      <th scope="col">부산</th>
      <th scope="col">제주</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">1분기</th>
      <td>1,000만원</td>
      <td>800만원</td>
      <td>500만원</td>
    </tr>
    <tr>
      <th scope="row">2분기</th>
      <td>1,200만원</td>
      <td>900만원</td>
      <td>600만원</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th scope="row">합계</th>
      <td>2,200만원</td>
      <td>1,700만원</td>
      <td>1,100만원</td>
    </tr>
  </tfoot>
</table>

```

### 4.5 접근성 있는 표 작성 지침

- 복잡한 표는 피하고 가능한 단순하게 구성합니다.
- 표 제목을 `<caption>`으로 명확히 제공합니다.
- 제목 셀에는 항상 `scope` 속성을 사용합니다.
- 복잡한 표에는 `id`와 `headers` 속성을 활용합니다.
- 빈 셀은 `&nbsp;`나 "없음"과 같은 텍스트로 채웁니다.
- 시각적 구분만을 위한 표 사용을 지양합니다.

## 5. 웹 접근성 개선 실전 기법

### 5.1 대체 텍스트 제공

| 요소 | 접근성 향상 방법 | 예시 |
| --- | --- | --- |
| 이미지 | `alt` 속성 추가 | `<img src="logo.png" alt="회사 로고">` |
| 인포그래픽 | 상세한 설명 제공 | `<img src="chart.png" alt="2025년 분기별 매출 증가 그래프" longdesc="chart-desc.html">` |
| 아이콘 | 의미 전달 | `<i class="icon-search" aria-hidden="true"></i><span class="sr-only">검색</span>` |
| 이미지 버튼 | 목적 설명 | `<input type="image" src="submit.png" alt="양식 제출">` |

### 5.2 키보드 접근성

| 문제 | 해결방법 | 코드 예시 |
| --- | --- | --- |
| 비활성화된 링크 | `href` 속성 추가 | `<a href="#" role="button" onclick="myFunc()">클릭</a>` |
| 사용자 정의 위젯 | ARIA 역할 추가 | `<div role="button" tabindex="0" onclick="toggle()">토글</div>` |
| 키보드 트랩 | 탭 순서 유지 | 모달 창에서 닫기 버튼으로 나갈 수 있게 설계 |
| 숨겨진 컨텐츠 | 적절한 포커스 관리 | `document.getElementById('hidden').focus()` |

### 5.3 명확한 레이블과 지침

| 요소 | 접근성 향상 방법 | 예시 |
| --- | --- | --- |
| 폼 필드 | 명시적 레이블 연결 | `<label for="name">이름</label><input id="name">` |
| 에러 메시지 | 명확한 오류 설명 | `<p aria-live="polite" role="alert">비밀번호는 8자 이상이어야 합니다</p>` |
| 필수 입력 필드 | 시각/텍스트로 표시 | `<label for="email">이메일 <span class="required">*</span></label>` |
| 진행 단계 | 현재 단계 표시 | `<ol class="progress"><li class="current">1. 정보 입력</li>...</ol>` |

### 5.4 색상과 대비

| 요구사항 | 접근성 기준 | 개선 방법 |
| --- | --- | --- |
| 텍스트 대비 | AA 기준: 4.5:1 이상 | 더 어두운 텍스트 또는 더 밝은 배경 사용 |
| 링크 구분 | 색상 외 추가 단서 | 밑줄, 아이콘, 굵게 등 추가 |
| 정보 전달 | 색상에만 의존 금지 | 아이콘, 패턴, 텍스트 레이블 추가 |
| 포커스 표시 | 명확한 시각적 구분 | `:focus { outline: 3px solid blue; }` |

### 5.5 ARIA(Accessible Rich Internet Applications)

| ARIA 유형 | 설명 | 예시 |
| --- | --- | --- |
| 역할(roles) | 요소의 기능 정의 | `<div role="button">클릭</div>` |
| 속성(properties) | 요소의 특성 설명 | `<input aria-required="true">` |
| 상태(states) | 요소의 현재 상태 | `<button aria-pressed="true">` |
| 라이브 리전 | 동적 콘텐츠 영역 | `<div aria-live="polite">` |

```html
<!-- ARIA 활용 예시: 커스텀 탭 컴포넌트 -->
<div role="tablist">
  <button role="tab" aria-selected="true" id="tab1">탭 1</button>
  <button role="tab" aria-selected="false" id="tab2">탭 2</button>
</div>
<div role="tabpanel" aria-labelledby="tab1">탭 1 내용</div>
<div role="tabpanel" aria-labelledby="tab2" hidden>탭 2 내용</div>

```

## 6. 레이아웃 발전 역사와 시맨틱 웹

### 6.1 웹 레이아웃의 진화

| 시대 | 특징 | 문제점 |
| --- | --- | --- |
| 초기 웹 | 단순 텍스트 나열 | 디자인 제한적 |
| 테이블 레이아웃 | 표를 이용한 배치 | 접근성 저하, 유지보수 어려움 |
| 프레임 레이아웃 | 여러 HTML 페이지 조합 | 검색엔진 최적화 문제, 사용성 저하 |
| div 중심 | CSS로 스타일링한 div | 의미론적 구조 부재 |
| 시맨틱 마크업 | 의미를 가진 HTML5 요소 | 접근성, SEO 개선 |

### 6.2 시맨틱 웹의 이점

| 이점 | 설명 |
| --- | --- |
| 검색엔진 최적화(SEO) | 검색엔진이 콘텐츠 의미와 중요도를 더 잘 이해 |
| 접근성 향상 | 보조 기술 사용자가 페이지 구조를 더 쉽게 파악 |

# 최종 요약: 웹 접근성과 시맨틱 HTML 핵심 정리

## 웹 접근성

- **정의**: 모든 사용자(장애인, 고령자 포함)가 웹 콘텐츠에 동등하게 접근할 수 있도록 보장
- **법적 근거**: 한국의 장애인차별금지법(2013년 4월부터 시행)에 따라 의무화
- **WCAG 4대 원칙**:
    1. 인식 가능(Perceivable): 모든 콘텐츠를 인식할 수 있어야 함
    2. 운용 가능(Operable): 인터페이스 요소를 조작할 수 있어야 함
    3. 이해 가능(Understandable): 정보와 작동 방식을 이해할 수 있어야 함
    4. 견고함(Robust): 다양한 기기와 보조 기술에서 접근 가능해야 함
- **준수 수준**: A(최소), AA(표준), AAA(최적)
- **접근성 확인 방법**: 키보드 탐색, 포커스 확인, 스크린 리더 테스트, 자동화 도구(Lighthouse 등) 활용

## 시맨틱 HTML

- **정의**: HTML 요소에 의미를 부여하여 콘텐츠의 구조와 의미를 명확히 전달하는 마크업 방식
- **이점**: SEO 개선, 접근성 향상, 코드 가독성 및 유지보수성 향상
- **주요 시맨틱 요소**:
    - `<header>`: 소개 및 탐색 요소
    - `<nav>`: 주요 탐색 링크
    - `<main>`: 페이지 주요 콘텐츠(페이지당 한 번만 사용)
    - `<article>`: 독립적인 콘텐츠
    - `<section>`: 관련 콘텐츠 그룹
    - `<aside>`: 부가 콘텐츠
    - `<footer>`: 페이지 또는 섹션의 마무리 정보

## HTML 표(Table)

- **기본 구조**: `<table>`, `<tr>`, `<td>`, `<th>`, `<caption>`
- **구조화 요소**: `<thead>`, `<tbody>`, `<tfoot>`, `<colgroup>`, `<col>`
- **접근성 속성**: `scope`, `colspan`, `rowspan`, `idheaders`
- **접근성 지침**: 단순한 구조 유지, 명확한 제목 제공, 제목 셀에 `scope` 속성 사용

## 접근성 개선 실전 기법

- **대체 텍스트**: 모든 이미지와 멀티미디어에 대체 텍스트 제공
- **키보드 접근성**: 모든 기능을 키보드로 이용 가능하게 구현
- **명확한 레이블**: 폼 요소에 명시적 레이블 연결
- **색상 대비**: 텍스트와 배경 간 충분한 대비(4.5:1 이상) 제공
- **ARIA 활용**: 표준 HTML만으로 접근성 확보가 어려운 경우 보완

## 레이아웃 발전과 현대적 접근

- **과거**: 테이블 레이아웃, 프레임, 의미 없는 div 사용
- **현재**: 시맨틱 요소, CSS Flexbox/Grid 활용
- **호환성**: 구형 브라우저 지원이 필요한 경우 대체 방법 고려

하나의 문장으로 요약하면: "웹 접근성과 시맨틱 HTML은 모든 사용자를 위한 포용적 웹 경험을 제공하고, 코드의 유지보수성 및 검색 엔진 최적화를 향상시키는 필수적인 웹 개발 원칙입니다."

# 웹 접근성과 시맨틱 HTML 복습 퀴즈

## 문제 1

**웹 접근성의 네 가지 기본 원칙(POUR) 중, "모든 사용자가 서비스의 콘텐츠, 기능 사용법 등을 이해하기 쉬워야 한다"는 원칙은 무엇인가요?**

A) 인식 가능(Perceivable)

B) 운용 가능(Operable)

C) 이해 가능(Understandable)

D) 견고함(Robust)

**정답: C) 이해 가능(Understandable)**

**해설:** WCAG(웹 콘텐츠 접근성 가이드라인)의 네 가지 원칙 중 "이해 가능(Understandable)"은 모든 사용자가 웹 콘텐츠와 인터페이스의 작동 방식을 이해할 수 있어야 한다는 원칙입니다. 이는 명확하고 일관된 탐색, 이해하기 쉬운 콘텐츠, 예측 가능한 기능 등을 포함합니다. "인식 가능"은 콘텐츠를 감지할 수 있어야 함을, "운용 가능"은 인터페이스를 조작할 수 있어야 함을, "견고함"은 다양한 환경과 기술에서 접근 가능해야 함을 의미합니다.

## 문제 2

**다음 중 독립적으로 배포하거나 재사용할 수 있는 콘텐츠를 마크업할 때 가장 적합한 HTML 요소는 무엇인가요?**

A) `<section>`

B) `<article>`

C) `<div>`

D) `<aside>`

**정답: B) `<article>`**

**해설:** `<article>` 요소는 독립적으로 구분하여 배포하거나 재사용할 수 있는 콘텐츠에 사용됩니다. 뉴스 기사, 블로그 포스트, 댓글, 위젯처럼 사이트의 다른 기능에 영향을 주지 않고 독립적인 기능을 수행하는 콘텐츠에 적합합니다. `<section>`은 관련된 콘텐츠를 그룹화할 때 사용하며, `<div>`는 의미적 가치 없이 스타일링을 위한 컨테이너로 사용됩니다. `<aside>`는 문서의 주요 내용과 간접적으로 연관된 부가 정보(예: 사이드바)에 사용됩니다.

## 문제 3

**HTML 표에서 열 헤더를 올바르게 지정하는 방법은 무엇인가요?**

A) `<th column="true">제목</th>`

B) `<td header="true">제목</td>`

C) `<th scope="col">제목</th>`

D) `<td type="header">제목</td>`

**정답: C) `<th scope="col">제목</th>`**

**해설:** HTML 표에서 열 헤더를 올바르게 지정하려면 `<th>` 요소와 함께 `scope="col"` 속성을 사용해야 합니다. 이 속성은 스크린 리더와 같은 보조 기술이 테이블의 구조를 이해하고 사용자에게 올바르게 정보를 전달하는 데 중요합니다. 특히 표의 각 셀이 어떤 헤더와 관련되어 있는지를 명확히 하여 접근성을 향상시킵니다. A, B, D 옵션은 HTML 표준에 존재하지 않는 속성이거나 올바른 마크업 방법이 아닙니다.

## 문제 4

**이미지의 접근성을 위해 가장 중요한 HTML 속성은 무엇인가요?**

A) `title`

B) `src`

C) `alt`

D) `aria-label`

**정답: C) `alt`**

**해설:** 이미지의 접근성을 위해 가장 중요한 HTML 속성은 `alt`(대체 텍스트) 속성입니다. 이 속성은 이미지를 볼 수 없는 시각 장애인 사용자에게 스크린 리더를 통해 이미지의 내용과 목적을 설명해줍니다. 또한 이미지 로딩에 실패했을 때 시각적으로도 표시됩니다. `title` 속성은 마우스 오버 시 추가 정보를 제공하지만 접근성의 핵심은 아닙니다. `src`는 이미지 파일의 경로를 지정하는 속성이며, `aria-label`은 주로 다른 HTML 요소에서 접근성 레이블을 제공하는 데 사용되지만 이미지에는 `alt` 속성이 우선적으로 사용됩니다.

## 문제 5

**다음 중 웹 접근성을 확인하기 위한 간단한 테스트 방법이 아닌 것은 무엇인가요?**

A) 키보드 탭으로 모든 기능을 탐색할 수 있는지 확인하기

B) 요소에 포커스 되었을 때 시각적 표시가 명확한지 확인하기

C) W3C 유효성 검사기로 HTML 코드 검증하기

D) 마우스로만 작동하는 커스텀 컨트롤 추가하기

**정답: D) 마우스로만 작동하는 커스텀 컨트롤 추가하기**

**해설:** 마우스로만 작동하는 커스텀 컨트롤을 추가하는 것은 웹 접근성을 확인하는 방법이 아니라, 오히려 접근성을 저해하는 행위입니다. 모든 기능은 키보드로도 접근 및 작동이 가능해야 합니다. 나머지 선택지는 모두 접근성을 확인하는 유효한 방법입니다: 키보드 탐색 테스트는 운용 가능성을 확인하고, 포커스 표시 확인은 시각적 인식 가능성을 테스트하며, W3C 유효성 검사는 코드의 표준 준수 여부를 확인하여 견고함을 높입니다.

## 문제 6

**WCAG 준수 수준 중, 전 세계 대부분의 접근성 규칙과 규정에서 기준으로 사용되는 수준은 무엇인가요?**

A) 레벨 A

B) 레벨 AA

C) 레벨 AAA

D) 레벨 AAAA

**정답: B) 레벨 AA**

**해설:** WCAG 준수 수준 중 레벨 AA는 전 세계 대부분의 접근성 규칙 및 규정에서 표준으로 사용되는 수준입니다. 레벨 A는 최소한의 접근성 요구사항만 충족하며, 이를 충족하지 않으면 일부 사용자가 웹사이트를 전혀 사용할 수 없을 수 있습니다. 레벨 AA는 중간 수준으로, 합리적인 접근성을 제공합니다. 레벨 AAA는 가장 높은 수준으로, 모든 사용자에게 최적의 접근성을 제공하지만 모든 콘텐츠 유형에 적용하기 어려울 수 있습니다. 레벨 AAAA는 WCAG 표준에 존재하지 않습니다.

## 문제 7

**다음 중 텍스트를 시각적으로 이탤릭체로 표시하면서도 강조의 의미를 전달하는 시맨틱 HTML 요소는 무엇인가요?**

A) `<i>`

B) `<em>`

C) `<italic>`

D) `<text style="font-style: italic">`

**정답: B) `<em>`**

**해설:** `<em>` 요소는 텍스트를 이탤릭체로 표시하면서 강조(emphasis)의 의미를 전달하는 시맨틱 HTML 요소입니다. 시각적 표현뿐만 아니라 스크린 리더와 같은 보조 기술에서도 강조로 인식됩니다. 반면 `<i>` 요소는 시각적으로는 이탤릭체로 표시되지만, 의미적으로는 단순히 통상적인 텍스트와 구분되는 부분(전문 용어, 외국어 구문 등)을 나타냅니다. `<italic>`은 존재하지 않는 태그이며, `<text style="font-style: italic">`은 올바른 HTML 요소가 아니라 스타일 속성을 잘못 적용한 예입니다.

## 문제 8

**다음 중 웹 페이지 내에서 한 번만 사용해야 하는 시맨틱 요소는 무엇인가요?**

A) `<header>`

B) `<footer>`

C) `<main>`

D) `<section>`

**정답: C) `<main>`**

**해설:** `<main>` 요소는 문서의 주요 콘텐츠 영역을 나타내며, HTML5 명세에 따라 웹 페이지 내에서 한 번만 사용해야 합니다. 이 요소는 사이드바, 탐색 링크, 저작권 정보, 로고 등 여러 페이지에 반복되는 콘텐츠를 포함해서는 안 됩니다. 반면 `<header>`, `<footer>`, `<section>`은 페이지 내에서 여러 번 사용할 수 있습니다. 예를 들어, 각 `<article>` 또는 `<section>` 내에서도 자체적인 `<header>`와 `<footer>`를 가질 수 있습니다.

## 문제 9

**다음 중 CSS background-image와 `<img>` 태그 중 선택할 때 고려해야 할 접근성 관점의 설명으로 가장 적절한 것은?**

A) CSS background-image가 항상 더 좋은 접근성을 제공한다

B) `<img>` 태그는 항상 CSS background-image보다 로딩이 느려 접근성에 좋지 않다

C) 콘텐츠에 의미가 있는 이미지는 `<img>`를, 장식용 이미지는 CSS background-image를 사용하는 것이 접근성에 좋다

D) 접근성을 위해서는 모든 이미지를 `<svg>` 요소로 변환해야 한다

**정답: C) 콘텐츠에 의미가 있는 이미지는 `<img>`를, 장식용 이미지는 CSS background-image를 사용하는 것이 접근성에 좋다**

**해설:** 접근성 관점에서 의미 있는 이미지(사용자의 이해에 중요한 정보를 전달하는 이미지)는 `<img>` 태그를 사용하고 적절한 대체 텍스트(`alt` 속성)를 제공해야 합니다. 반면, 순수하게 장식 목적인 이미지는 CSS background-image로 처리하는 것이 좋습니다. CSS 배경 이미지는 스크린 리더에 의해 무시되므로, 스크린 리더 사용자에게 불필요한 정보를 전달하지 않습니다. 속도나 SVG 변환은 접근성의 주요 고려사항이 아닙니다.

# 💭 오늘의 회고

**배운 점**

- **colgroup**과 **col**을 활용해 열 스타일을 지정한다.
- 단순한 표에는 **scope**를, 병합되거나 복잡한 표에는 **id-headers**를 활용한다.
- 시멘틱 태그는 가독성, 유지보수 및 SEO 향상을 위해 사용한다.
- **article**은 자체적으로 의미 있는 콘텐츠에, **section**은 페이지 내 연관 주제에 사용한다.
- **button**은 팝업 창이나 특정 반응을 유도할 때 적합하며, **a** 태그는 로그인 화면처럼 다른 페이지로 이동할 때 활용한다.
- 보안이 중요한 메일 시스템 등에는 오랜 방식인 **table** 구조가 여전히 사용된다.
